Process Descriptions

Brief description of each numbered process in the Level 1 DFD.

Process Description Actors
1.0 Attendance Recording Handles employee time-in/time-out, computes hours worked, determines status (present/late/absent/half-day), supports manual corrections and auto-absent marking. Employee, Branch Manager
2.0 Leave Management Processes leave requests, tracks leave balances, handles approval/denial workflow, updates attendance records when leave is approved. Employee, HR, Branch Manager
3.0 Employee Management Manages employee profiles including personal info, employment type, branch and shift assignments, and optional user account creation. HR, Admin
4.0 User & Role Management Manages system accounts, assigns Spatie roles (admin, hr, branch_manager, employee), controls permission-based access across all modules. Admin
5.0 Reporting & Analytics Generates attendance reports (daily/monthly/custom range), computes analytics metrics (absenteeism rate, peak days, trends), and produces exportable outputs. HR, Admin, Branch Manager
6.0 Forecasting Engine Applies Holt-Winters Triple Exponential Smoothing on historical attendance data to predict future absenteeism per branch. Falls back to moving average when data is insufficient (<14 tracked days). Branch Manager, Admin, Automated
7.0 Notification Service Dispatches in-app and email notifications for leave events (submitted, approved, denied), account creation, and attendance alerts to relevant actors. All actors
8.0 System Configuration Manages global settings including organization info, late/absent thresholds, leave quotas, Holt-Winters smoothing parameters (α, β, γ), and alert email recipients. Admin
